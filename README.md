# Информационная система учета МТБ колледжа

Веб-приложение для учета материально-технической базы колледжа.

## Функционал

### Основные модули:
- **Оборудование** - учет всего оборудования с инвентарными номерами
- **Помещения** - управление аудиториями, лабораториями, складами
- **Сотрудники** - база ответственных лиц
- **История изменений** - полная история всех изменений оборудования
- **Инвентаризация** - проведение и контроль инвентаризаций
- **Списание** - учет списанного оборудования с возможностью восстановления

### Возможности:
- Отслеживание перемещений оборудования
- История смены ответственных лиц
- Учет изменения состояния оборудования
- Просмотр детальной информации о помещении (все оборудование внутри)
- Просмотр детальной информации о сотруднике (за что отвечает)
- Списание и восстановление оборудования
- Статистика на главной странице (дашборд)

## Технологии

- **Backend**: PHP 7.4+
- **База данных**: MySQL 5.7+ / MariaDB 10.3+
- **Frontend**: React 18, TailwindCSS, Font Awesome

## Установка

### 1. Требования
- PHP 7.4 или выше
- MySQL 5.7 / MariaDB 10.3 или выше
- Веб-сервер (Apache/Nginx)

### 2. Настройка базы данных

```bash
# Войдите в MySQL
mysql -u root -p

# Выполните скрипт создания БД
source /путь/к/проекту/database.sql
```

Или через phpMyAdmin - импортируйте файл `database.sql`

### 3. Настройка подключения

Отредактируйте файл `config/database.php`:

```php
define('DB_HOST', '127.0.0.1');  // Хост БД
define('DB_PORT', '3306');       // Порт
define('DB_NAME', 'college_mtb'); // Имя БД
define('DB_USER', 'root');       // Пользователь
define('DB_PASS', '');           // Пароль
```

### 4. Запуск

Скопируйте файлы в директорию веб-сервера:
- Для XAMPP: `C:\xampp\htdocs\mtb\`
- Для OpenServer: `domains\mtb\`

Откройте в браузере: `http://localhost/mtb/`

## Тестовые данные

После выполнения `database.sql` в системе появятся демо-данные:
- 2 пользователя (admin/123456, user1/123456)
- 8 категорий оборудования
- 5 статусов состояния
- 5 сотрудников
- 8 помещений
- 13 единиц оборудования
- Примеры истории изменений

## Схема базы данных

### Основные таблицы:
- `users` - пользователи системы
- `categories` - категории оборудования
- `condition_status` - статусы состояния
- `premises` - помещения
- `employees` - сотрудники
- `equipment` - оборудование
- `equipment_history` - история изменений
- `movements` - перемещения (для совместимости)
- `inventories` - инвентаризации
- `inventory_items` - позиции инвентаризации
- `write_offs` - акты списания
